1 Test Strategy

The MOOVIZ test strategy follows a pyramid approach, emphasizing automated tests at lower levels and targeted manual/E2E tests for critical user flows. All tests are designed to validate functionality, security, and performance across the full stack.

Unit Tests

Tool: Jest + React Testing Library
Scope: Pure functions, hooks, utilities, validators, state machine logic
Coverage target: 80% of business logic

Integration Tests

Tool: Firebase Emulator Suite
Scope: Cloud Functions, Firestore triggers, security rules, FCM handlers
Coverage target: All state transitions + RBAC

E2E Tests

Tool: Detox (mobile), Playwright (admin)
Scope: Full user flows on iOS/Android devices and admin web panel
Coverage target: All critical paths

Security Tests

Tool: Custom scripts + Firebase rules testing
Scope: Penetration testing checklist, RBAC bypass attempts, injection vectors
Coverage target: OWASP Mobile Top 10

Performance Tests

Tool: Artillery / k6
Scope: Load testing Cloud Functions, concurrent Firestore operations, geohash queries
Coverage target: P95 latency thresholds

4 Test Schedule

Total testing duration: 7 working days, structured in four phases. Each phase builds on the previous, with blocking issues resolved before advancing.

Phase 1: Unit + Integration Tests

Days 1-2 (2 days)

Execute all unit tests (validators, state machine, utilities) and integration tests against Firebase Emulator. Covers: AUTH-05, AUTH-08, DEL-03 through DEL-14, SEC-01 through SEC-11, CHAT-02, CHAT-03, NOTIF-01 through NOTIF-08.

Phase 2: E2E on Emulator/Simulator

Days 3-4 (2 days)

Run Detox E2E tests on iOS Simulator and Android Emulator. Covers full user flows: AUTH-01 through AUTH-04, AUTH-06, AUTH-07, DEL-01, DEL-02, CHAT-01. Admin panel tests via Playwright: ADMIN-01 through ADMIN-06.

Phase 3: E2E on Physical Devices

Days 5-6 (2 days)

Repeat critical E2E flows on physical iPhone and Android device with Hebrew locale. Focus on RTL layout validation, OTP via real SMS, push notification delivery, camera/gallery for proof photos, GPS location tracking, and offline/online transitions.

Phase 4: Security + Performance

Day 7 (1 day)

Execute security penetration checklist (SEC-01 through SEC-11) against staging environment. Run load tests (PERF-01 through PERF-04) against staging Firebase project. Document findings and create remediation tickets for any issues.

Defect Management

All defects are logged in the CRM system with severity (Critical/High/Medium/Low), assigned to the responsible developer, and tracked through resolution. Critical and High defects block release; Medium defects may be deferred with CTO approval. Regression tests are executed after each fix to confirm resolution without side effects.
