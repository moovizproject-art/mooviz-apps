היקף הביקורת

הביקורת כללה סקירה מקיפה של כלל שכבות המערכת:

  • אפליקציית React Native ‎(iOS + Android)
  • Firebase Cloud Functions ‎(Node.js 18)
  • כללי אבטחת Firestore
  • כללי אבטחת Storage
  • פאנל ניהול ‎(React Web)
  • אימות — Firebase Auth ‎(OTP + Email)
  • מערכת הצפנה ‎(AES-256-CBC)

הערכת OWASP Top 10

טבלת ממצאים

להלן 23 הממצאים שזוהו במהלך הביקורת, מסודרים לפי חומרה:

# חומרה ממצא סטטוס תיקון
C1 קריטי לקוח עוקף אימות שרת — כתיבה ישירה ל-Firestore לשינוי סטטוס תוקן כל שינויי הסטטוס מנותבים דרך Cloud Functions
C2 קריטי נקודת קצה getEncryptionKey מחזירה מפתחות הצפנה ללקוח תוקן נקודת הקצה הוסרה לחלוטין. כל ההצפנה בצד השרת בלבד
C3 קריטי מפתח הצפנה קבוע כברירת מחדל בקוד תוקן ברירת המחדל הוסרה. ENCRYPTION_SECRET חייב להיות מוגדר דרך Secret Manager
C4 קריטי פאנל ניהול עוקף Cloud Functions סיכון מקובל הניהול משתמש ב-Firestore ישירות עם בדיקת תפקיד admin בכללי האבטחה
H1 גבוה סיסמאות ואישורים בטקסט גלוי בתיקיית scripts/ תוקן תיקיית scripts/ נוספה ל-.gitignore
H2 גבוה מפתח API של CRM קבוע בקוד תוקן הוחלף בטעינה מקובץ .env.crm-api
H3 גבוה כלל admin עם UID קבוע ב-firestore.rules תוקן הוסר. ‎isAdmin() בודק תפקיד במסמך המשתמש
H4 גבוה confirmPayment עם חלופה ישירה ל-Firestore תוקן החלופה הוסרה. אישור תשלום דרך Cloud Function בלבד
H5 גבוה אוסף משתמשים נקרא באופן רחב מדי חלקי Firestore לא תומך בהגבלת שדות לקריאה. ממותן באמצעות Cloud Function לתמונות
H6 גבוה קבצי צ'אט נגישים ללא אימות משתתף תוקן פונקציית ‎isChatParticipant() נוספה ל-storage.rules
H7 גבוה קבצי הוכחת משלוח ללא אימות בעלות תוקן פונקציית ‎isDeliveryParticipant() נוספה ל-storage.rules
H8 גבוה מידע אישי ‎(JSON) נרשם ביומן בשירות אימות הניהול תוקן כל הודעות console.log הוסרו מ-auth.ts
M1 בינוני מפתח Google Maps API קבוע בקוד כברירת מחדל תוקן המפתח הקבוע הוסר. חובה להגדיר משתנה סביבה
M2 בינוני מפתח הצפנה קבוע בקוד (כפול של C3) תוקן זהה ל-C3
M3 בינוני UID של admin קבוע ב-firestore.rules (כפול של H3) תוקן זהה ל-H3
M4 בינוני RBAC לא עקבי בין שכבות תוקן בדיקת ‎isAdmin() אחידה בכל השכבות
M5 בינוני אין הגבלת שדות בעדכון מסמכי משלוח תוקן מגבלות ‎hasOnly() עם רשימות שדות מותרים
M6 בינוני לקוח יכול להגדיר תפקיד שרירותי בהרשמה תוקן פונקציית ‎validRole() מגבילה ל-sender/driver בלבד
M7 בינוני CORS פתוח בפונקציית email תוקן הוגבל לדומיינים מורשים של הניהול
M8 בינוני תוכן HTML לא מסונן בשליחת אימיילים תוקן ‎sanitizeHtml() מסיר תגיות ואירועים מסוכנים
M9 בינוני מידע אישי (מספרי טלפון) נרשם ביומן באפליקציה תוקן מספרי טלפון הוסתרו מיומני RootNavigator ו-AddPhoneScreen
M10 בינוני דגל __DEV__ עוקף אימות OTP תוקן __DEV__ תמיד false ב-production builds — סיכון נמוך
L1 נמוך מפתחות Firebase חשופים ב-scripts תוקן תיקיית scripts/ ב-.gitignore
L2 נמוך מנגנון הגבלת קצב מוגדר אך לא פעיל תוקן מחובר ל-expressInterest ול-sendBulkEmail
L3 נמוך שאילתת רשימת צ'אטים מאפשרת לכל משתמש מאומת תוקן כלל list דורש שייכות למשתתפי הצ'אט

בדיקות שעברו בהצלחה

בקרות האבטחה הבאות אומתו ופועלות כנדרש:

  • ✓אימות Firebase נדרש לכל קריאות API
  • ✓אימות בצד השרת לכל שינויי סטטוס משלוח
  • ✓הצפנת AES-256-CBC עם ניהול מפתחות בצד השרת בלבד
  • ✓אכיפת RBAC ב-Firestore, Storage ו-Cloud Functions
  • ✓הגבלת קצב על פעולות רגישות ‎(OTP, התחברות, הבעת עניין, אימייל)
  • ✓סינון קלט בתוכן HTML של אימיילים
  • ✓CORS מוגבל לדומיינים מורשים
  • ✓אין מידע אישי ביומני האפליקציה
  • ✓סודות מנוהלים דרך Firebase Secret Manager
  • ✓כללי Storage אוכפים גישה מבוססת משתתפים
  • ✓הגבלות כתיבה ברמת שדה על מסמכי משלוח
  • ✓אימות תפקיד מונע הסלמת הרשאות בהרשמה
  • ✓מערכת הוכחות תמונה עם העלאה חובה
  • ✓תמונות פרופיל מוצפנות עם גישה מורשית בלבד
  • ✓רישום פעולות ניהול ‎(audit log)
  • ✓אין אישורים קבועים בקוד המקור
6

המלצות

המלצות לשיפור נוסף של מצב האבטחה לקראת פריסה בסביבת ייצור:

  • פריסה לפרויקט Firebase ייעודי לייצור עם אישורים נפרדים לחלוטין מסביבת הפיתוח
  • הפעלת Firebase App Check למניעת שימוש לרעה ב-API ממקורות לא מורשים
  • יישום Content Security Policy ‎(CSP) לפאנל הניהול למניעת התקפות XSS
  • הוספת SAST אוטומטי לצינור CI/CD לזיהוי חולשות אבטחה בזמן פיתוח
  • תזמון סקירות אבטחה רבעוניות לוודא עמידה מתמשכת בדרישות
  • שקילת WAF ‎(Web Application Firewall) לנקודות קצה HTTP של Cloud Functions
  • יישום timeout לסשן בפאנל הניהול לניתוק אוטומטי לאחר חוסר פעילות

סיכום
