7. Gap Analysis

GAP-1: No Formal Information Security Policy HIGH

TSC: CC1, CC2 | Current: Security requirements in CLAUDE.md and audit reports | Gap: No standalone, board-approved information security policy document | Recommendation: Create formal ISMS policy aligned with ISO 27001 A.5.1

GAP-2: No Privacy Policy / Data Processing Notice HIGH

TSC: Privacy (PV.2, PV.3) | Current: No published privacy policy | Gap: Israeli Privacy Protection Law requires data processing notice; App store submission requires privacy policy URL | Recommendation: Draft and publish privacy policy in Hebrew and English before production launch

GAP-3: No Data Subject Rights Implementation HIGH

TSC: Privacy (PV.3) | Current: No self-service data export or deletion | Gap: Must support data access requests and right to deletion | Recommendation: Implement Cloud Function for user data export and account deletion workflow

GAP-4: No Incident Response Plan MEDIUM

TSC: CC7.3 | Current: Ad-hoc remediation demonstrated in security audit | Gap: No documented incident response plan with roles, escalation, and communication procedures | Recommendation: Document IRP covering detection, containment, eradication, recovery, and lessons learned

GAP-5: No External Penetration Testing MEDIUM

TSC: CC3, CC4 | Current: Internal security audit only | Gap: No third-party penetration test performed | Recommendation: Engage external security firm for penetration test before production launch

GAP-6: Production Environment Not Yet Deployed MEDIUM

TSC: CC8.3, AV | Current: Only dev Firebase project active (mooviz-app-9b766) | Gap: Production Firebase project (mooviz-prod) pending billing setup | Recommendation: Complete production environment setup with separate security rules and secrets

GAP-7: Remaining Security Audit Findings MEDIUM

TSC: CC5 | Current: Finding C1 (client bypasses) partially fixed; L3 (chat list query filtering) low risk | Gap: Some status transitions still allow client-side update via Firestore rules | Recommendation: Migrate all status transitions exclusively to Cloud Functions (C1 complete fix)

GAP-8: No Data Classification Policy LOW

TSC: CF.5 | Current: Implicit classification via encryption tiers | Gap: No formal data classification document | Recommendation: Create data classification matrix covering public, internal, confidential, and restricted categories

GAP-9: No Formal Backup/DR Strategy LOW

TSC: AV.5 | Current: Google-managed Firestore PITR | Gap: No documented application-level backup and disaster recovery strategy | Recommendation: Document RTO/RPO targets and backup verification procedures

8. Remediation Roadmap

Priority-ordered remediation items for achieving full SOC 2 Type I compliance. Items are sequenced to address highest-risk gaps first while aligning with the M3/M4 milestone timeline.
1
Privacy Policy and Terms of Service HIGH

Draft and publish privacy policy (Hebrew + English) covering data collection, processing, retention, and user rights. Required for App Store / Play Store submission (M4 milestone). Timeline: 1 week.

2
Data Subject Rights (Access, Deletion, Portability) HIGH

Implement Cloud Function exportUserData and deleteUserAccount for GDPR/Israeli privacy law compliance. Include Firestore, Storage, and Auth data. Timeline: 1 week.

3
Formal Information Security Policy HIGH

Create standalone ISMS policy document covering scope, objectives, roles, and responsibilities. Align with ISO 27001 A.5.1. Timeline: 3 days.

4
Migrate All Status Transitions to Cloud Functions MEDIUM

Complete finding C1 remediation: remove client-side status write permissions from firestore.rules. All delivery status changes must go through deliveryCallable.ts. Timeline: 3 days.

5
Production Environment Setup MEDIUM

Activate mooviz-prod Firebase project with separate Firestore rules, Storage rules, and Secret Manager secrets. Configure CI/CD pipeline for staging and production deployment. Timeline: 2 days.

6
Incident Response Plan MEDIUM

Document IRP covering: detection, classification, escalation, containment, eradication, recovery, post-incident review. Define roles and communication channels. Timeline: 2 days.

7
External Penetration Test MEDIUM

Engage third-party security firm for penetration testing of Cloud Functions API, Firestore rules, and Storage rules. Scope: API fuzzing, authentication bypass, privilege escalation. Timeline: 2-3 weeks (vendor-dependent).

8
Data Classification and Backup Documentation LOW

Create data classification matrix and document backup/DR strategy with RTO/RPO targets. Verify Firestore PITR configuration. Timeline: 2 days.

9
PII Audit in Cloud Function Logs LOW

Audit all console.error and console.log calls in Cloud Functions for potential PII leakage (UIDs, emails). Replace with structured logging using redacted identifiers. Timeline: 1 day.
